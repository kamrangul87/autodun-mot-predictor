Autodun MOT Predictor – Model Details

← Back to MOT Predictor

Overview

This tool estimates the probability that a vehicle will pass its next MOT test. It is designed as an early warning and decision-support tool, not as a replacement for a professional inspection.

Current Model (v1)

  • Model type: Logistic regression
  • Features used:
    • Vehicle age (years since first registration)
    • Mileage (current odometer reading)
  • Training data: DVSA-style MOT samples (pass/fail outcomes)
  • Deployment: JSON model on GitHub → API on Vercel

How the Probability Is Calculated

The model computes a score z as a linear combination of the inputs: 

z = w_age * vehicle_age + w_mileage * mileage + intercept
probability = 1 / (1 + exp(-z))

The current parameters are stored in ml/models/mot_model_v1.json. For example: 

{
  "coef": [[-0.25, -0.00001]],
  "intercept": [3.0],
  "columns": ["vehicle_age", "mileage"]
}

Risk Bands

For the current version, probabilities are mapped to risk bands as follows:

  • Low risk: probability ≥ 0.80 – likely to pass
  • Medium risk: 0.50 ≤ probability < 0.80 – some issues possible
  • High risk: probability < 0.50 – higher chance of failure

Limitations

  • Only uses age and mileage (no repair history, no detailed defect list yet).
  • Based on historical MOT-style data; future conditions may differ.
  • Does not guarantee a pass or fail – it is a statistical estimate.

Planned Improvements

  • Add more features (make, model, fuel type, previous failures, etc.).
  • Train on larger UK-wide MOT datasets.
  • Switch to more advanced models (e.g. gradient boosting) if needed.
  • Expose versioned metrics (accuracy, precision, recall) on an admin page.
